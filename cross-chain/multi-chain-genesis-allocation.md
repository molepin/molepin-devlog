# Multi-Chain Genesis Allocation

The earliest MOLEPIN deployment model was designed around chain-specific ecosystem assumptions.

The initial idea was to distribute supply according to:

- ecosystem activity
- liquidity potential
- bridge usage
- infrastructure growth
- future scalability

Initial allocation model:

| Chain | Allocation |
|---|---:|
| Ethereum | 3% |
| Base | 21% |
| Polygon | 7% |
| BNB Chain | 15% |
| Arbitrum | 25% |
| Optimism | 5% |
| Avalanche | 5% |
| zkSync Era | 10% |
| Klaytn | 3% |
| TRON | 5% |
| Reserve | 1% |

The important realization was that initial allocation does not define permanent liquidity dominance.

Bridge movement allows liquidity distribution to evolve dynamically over time.

This became one of the core design principles behind the MOLEPIN ecosystem.
